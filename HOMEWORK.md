# Homework: Kafka Connect S3 Source with Data Masking on AWS EKS

**Repository:** https://github.com/topsportas/M11_KafkaConnect_JSON_AWS

---

## Overview

This homework demonstrates how to use **Kafka Connect S3 Source Connector** to stream Avro-formatted hotel booking data from AWS S3 into a Kafka topic, with **data masking** applied to sensitive date fields using a Single Message Transform (SMT).

### Architecture

```
AWS S3 Bucket (Avro files)
        |
        v
Kafka Connect (S3SourceConnector)
   + MaskField SMT (masks date fields)
        |
        v
Kafka Topic: expedia  (running on EKS)
        |
        v
Confluent Control Center (monitoring)
```

## Docker Image

The custom Kafka Connect image (`connectors/Dockerfile`) extends the Confluent base image and installs both the S3 Sink and S3 Source connectors:

```dockerfile
FROM confluentinc/cp-server-connect:7.8.1 AS base
USER root
RUN confluent-hub install --no-prompt confluentinc/kafka-connect-s3:10.5.3
RUN confluent-hub install --no-prompt confluentinc/kafka-connect-s3-source:3.0.11
USER appuser
```

> **Note:** `kafka-connect-s3:10.5.3` only includes the **Sink** connector. The **Source** connector (`S3SourceConnector`) is a separate package (`kafka-connect-s3-source:3.0.11`) and must be installed explicitly.

The image is pushed to AWS ECR and referenced in `confluent-platform.yaml`.

## S3 Source Connector with Data Masking

The connector is registered via the Kafka Connect REST API. The config is generated by Terraform (`terraform/main.tf`) and saved to `terraform/aws-source-cc.json`.

The full connector config including the masking transform:

```json
{
  "name": "expedia",
  "config": {
    "connector.class": "io.confluent.connect.s3.source.S3SourceConnector",
    "s3.bucket.name": "kafka-connect-{suffix}",
    "s3.region": "us-west-1",
    "tasks.max": "3",
    "format.class": "io.confluent.connect.s3.format.avro.AvroFormat",
    "bootstrap.servers": "kafka:9071",
    "topics": "expedia",
    "topics.dir": "topics",
    "transforms": "maskDates",
    "transforms.maskDates.type": "org.apache.kafka.connect.transforms.MaskField$Value",
    "transforms.maskDates.fields": "date_time,srch_ci,srch_co"
  }
}
```

Registered with:

```bash
curl -s -X POST -H "Content-Type:application/json" \
  --data @aws-source-cc.json \
  http://localhost:8083/connectors
```

## Data Masking

### What is MaskField SMT?

`MaskField$Value` is a built-in Kafka Connect Single Message Transform that replaces specified field values with a type-safe "zero" value — empty string `""` for string fields, `null` for nullable fields.

### Why mask date fields?

Date fields like `date_time`, `srch_ci` (check-in), and `srch_co` (check-out) can reveal user booking patterns and are considered sensitive. Masking them at the connector level ensures they are never written to Kafka in clear text.

### Before masking:

```json
{
  "id": 649959,
  "date_time": "2015-05-03 11:23:55",
  "srch_ci": "2017-08-18",
  "srch_co": "2017-08-19",
  ...
}
```

### After masking:

```json
{
  "id": 2182172,
  "date_time": "",
  "srch_ci": "",
  "srch_co": "",
  ...
}
```

## S3 Data Structure

Avro files uploaded to S3 follow this folder structure expected by the connector:

```
kafka-connect-{suffix}/
└── topics/
    └── expedia/
        ├── partition=0/
        │   └── expedia+0+0000000000.avro
        ├── partition=1/
        │   └── expedia+1+0000000000.avro
        └── partition=2/
            └── expedia+2+0000000000.avro
```
